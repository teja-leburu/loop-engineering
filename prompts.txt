[STEP 1 / STAGE 1 — SPECIFY]
--------------------------------------------------------------------------------
Prompt: "Write SPEC.md for the base system: a server-authoritative chess web
app. Node/Express backend with a hand-written chess engine (no chess libraries,
because step 2 needs to inject custom rule modifiers into move generation),
vanilla JS frontend with a futuristic neon theme, in-memory game store, REST
API for create/state/legal-moves/move. Full rules: castling, en passant,
promotion, check/checkmate/stalemate/insufficient material. Include functional
requirements, quality requirements (node:test coverage), and acceptance
criteria a TA can verify."

--------------------------------------------------------------------------------
[STEP 1 / STAGE 2 — PLAN]
--------------------------------------------------------------------------------
Prompt: "Read SPEC.md and produce PLAN.md: an ordered checklist of small,
independently testable build tasks with '- [ ]' checkboxes. Each task must name
the files it touches and the tests that prove it. The build loop will execute
one task per iteration, so tasks must be small and ordered by dependency."

--------------------------------------------------------------------------------
[STEP 1 / STAGE 3 — BUILD (LOOP)]
--------------------------------------------------------------------------------
Loop prompt, given verbatim to the agent EVERY iteration until PLAN.md had no
unchecked boxes left (ralph-wiggum-style fixed prompt, state lives in the repo):

  "You are in a build loop. Read SPEC.md and PLAN.md. Find the FIRST unchecked
  '- [ ]' task in PLAN.md. Implement ONLY that task. Run 'npm test' and fix
  failures until green. Mark the task '- [x]' in PLAN.md. Commit everything
  with message 'loop(step1-build): iteration <N> — <task title>'. Do not start
  the next task. If all tasks are checked, output DONE."

(Iterations actually executed are visible in git history as
 loop(step1-build) commits.)

--------------------------------------------------------------------------------
[STEP 2 / STAGE 1 — SPECIFY]
--------------------------------------------------------------------------------
Prompt: "Write SPEC-EXTENSION.md for the Rule Forge extension: composable rule
modifiers a player toggles when creating a game — royal-knights (knights also
step like kings), berserker-pawns (pawns may capture straight ahead),
wormholes (paired teleport squares in the center), atomic-captures (captures
explode adjacent non-pawns), king-dash (king may move two squares in a line).
Modifiers hook into the engine's move generation / post-move pipeline, combine
freely, and check/checkmate must remain correct under any combination. Frontend
gets a 'Forge your variant' panel with toggles and visual cues (wormhole glow,
explosion flash). Include API changes and tests required."

[STEP 2 / STAGE 3 — BUILD (LOOP)]
